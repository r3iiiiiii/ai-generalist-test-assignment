# Тестовое задание

Сделал один цельный сайт, в котором объединил обе части задания:

- верхний блок это короткое портфолио под роль AI Generalist / AI Automation Specialist
- нижний блок это интерактивный прототип сервиса для поиска и сравнения поставщиков продуктов питания

Такой формат выбрал специально: его проще показывать на созвоне, и он лучше передает не только идею, но и то, как пользователь реально будет работать с сервисом.

## Что внутри

- блок с опытом, стеком и проектами, которые ближе всего к вакансии
- отдельный блок с логикой, как я обычно довожу AI-задачу до рабочего результата
- сервис поиска поставщиков по категории, региону, наличию доставки и документов
- готовые сценарии закупки
- настройка приоритетов выбора: цена, скорость, надежность
- сортировка по рекомендации, цене, минимальному заказу, скорости и названию
- короткий рейтинг, который помогает понять, с кем лучше связаться в первую очередь
- сравнение до 3 поставщиков
- заметки по каждому поставщику, сохраняются в браузере

Важно:

- текущая версия сервиса это именно интерактивный прототип
- для демонстрации логики я использовал ручной демо-датасет поставщиков
- в реальной реализации следующий шаг был бы в подключении внешних источников или внутренней базы закупки

## Как запустить

Самый простой вариант через Python:

```bash
python -m http.server 8000
```

Потом открыть в браузере:

```text
http://localhost:8000
```

Если команда `python` не сработает, можно использовать полный путь к Python или открыть `index.html` напрямую в браузере. Но для демонстрации лучше через локальный сервер.

## Как показывать на созвоне

1. Сначала коротко пройтись по верхнему блоку и показать, почему опыт релевантен роли.
2. Потом перейти к сервису и выбрать один из сценариев закупки.
3. Показать, что можно менять не только фильтры, но и приоритеты выбора.
4. Выбрать категорию, например `Упаковка` или `Ингредиенты`.
5. Ограничить регион, например `Екатеринбург`.
6. Добавить 2-3 поставщиков в сравнение.
7. Показать, как меняется рекомендация и как сохраняются заметки.

## Что можно сказать про идею

Смысл сервиса в том, чтобы менеджер по закупкам или операционный специалист не собирал все вручную по разным сайтам и таблицам. В одном месте он получает:

- быстрый фильтр по базовым параметрам
- переключение между разными сценариями закупки
- ручную настройку того, что важнее в текущем случае
- короткую сводку по поставщику
- контакты и источник
- простую подсказку по приоритету для первого контакта

Почему прототип именно такой:

- в рамках тестового важнее было показать понятную логику продукта и сценарий использования
- поэтому я сделал работающий интерфейс, где видно, как пользователь фильтрует, сравнивает и выбирает поставщика
- следующая итерация после такого прототипа это уже подключение реальных поставщиков и автоматизация сбора данных

## Технически

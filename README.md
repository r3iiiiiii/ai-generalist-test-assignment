# Тестовое задание

В репозитории одна цельная демо-страница, где я объединил обе части задания:

- верхняя часть это короткое портфолио под роль AI Generalist / AI Automation Specialist
- нижняя часть это интерактивный прототип сервиса для поиска и сравнения поставщиков продуктов питания

Мне показалось, что такой формат лучше обычного набора отдельных файлов. Его проще показать на созвоне, и он сразу дает понять не только опыт, но и то, как я думаю о продукте и интерфейсе.

## Что есть в демо

- блок с релевантным опытом, стеком и кейсами
- отдельный блок с реальными проектами и учебными задачами
- поиск поставщиков по категории и региону
- фильтры по доставке и документам
- готовые сценарии закупки
- настройка приоритетов выбора: цена, скорость, надежность
- сравнение поставщиков
- заметки по карточкам

## Важно

Сервисная часть здесь сделана как прототип. Для демонстрации логики я использовал ручной набор карточек поставщиков. Если развивать решение дальше, следующий шаг это подключение реальных источников, внутренней таблицы закупки или автоматизированного сбора данных.

## Запуск

Открыть PowerShell в папке проекта и выполнить:

```bash
python -m http.server 8000
```

Потом открыть в браузере:

```text
http://localhost:8000
```

Если `python` не срабатывает, можно попробовать:

```bash
py -m http.server 8000
```

## Почему сделал именно так

В рамках тестового мне хотелось показать не только визуальную часть, но и сам сценарий использования. Поэтому я сделал не просто список поставщиков, а прототип, где можно фильтровать, сравнивать и принимать решение, с кем связываться в первую очередь.

## Технически

Проект сделан на чистом HTML, CSS и JavaScript без лишних зависимостей. Это специально: так проще запускать, показывать и дорабатывать.
